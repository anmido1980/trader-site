# Правила работы с проектом Site

## Контекст

Проект: лендинг для публикации торговой статистики, результатов бэктестов и подписки на сигналы.

## Рабочая папка

Все файлы создаются и редактируются только внутри `d:\Yandex.Disk\Project\Claude\Site`.

## Организация корневой папки

- Корень содержит только конфигурацию проекта (`astro.config.mjs`, `package.json`, `tsconfig.json`), `README.md`, `.gitignore` и служебные папки (`.claude`, `.astro`, `dist`, `node_modules`, `posts`).
- Все изображения и медиа хранятся в:
  - `public/images/` — контент, который публикуется на сайте;
  - `assets/images/` — исходники, черновики и неиспользуемые изображения.
- Временные, лог- и кеш-файлы не остаются в корне.

## Правила нейминга

По умолчанию:

```text
YYYY-MM-DD_[тема]_[тип-файла].[расширение]
```

Где `[тип-файла]` отражает роль файла:

- `_post.json` — пост новостей/аналитики;
- `_original.jpg|png` — исходное изображение;
- `_hero.jpg|png` — изображение для Hero-секции;
- `_spec.md` — спецификация;
- `_backtest.png` — скриншот бэктеста.

Примеры:

- `posts/2026-06-21_future-of-agriculture_post.json`
- `assets/images/2022-12-09_backtest-screenshot_original.png`

Технические документы в `docs/` допускаются без даты, но с чётким семантическим именем (`architecture.md`, `tech-stack.md`). Карта документов — в `docs/README.md`.

## Обязательные файлы

- `README.md` в корне — стек, команды, структура папок, ссылки на документацию.
- `docs/README.md` — карта технической документации.

## .gitignore

Должен покрывать:

- `dist/`
- `.astro/`
- `node_modules/`
- `.env*`
- `.vscode/`, `.idea/`
- `.claude/settings.local.json`
- лог-файлы npm/yarn/pnpm

## Данные

- Числовые данные, списки сделок, кейсы, отзывы — в JSON-файлах (`src/data/`, `posts/`).
- Placeholder-значения в данных разрешены, но должны быть помечены комментарием или заменены перед публикацией.

## Разработка

- Команды:
  - `npm run dev` — локальный сервер;
  - `npm run build` — production-сборка;
  - `npm run check` — проверка типов Astro;
  - `npm run format` — форматирование Prettier.
- Перед коммитом/синхронизацией: `npm run check` и `npm run build` должны проходить без ошибок.

## Границы

- Не удалять файлы без явного разрешения.
- Не выходить за пределы рабочей папки проекта.
- Не добавлять в корень большие бинарные файлы без структурированного размещения.
