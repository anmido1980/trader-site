## Порядок чтения для нового участника

1. `architecture.md` — понять структуру.
2. `tech-stack.md` — понять стек.
3. `spec.md` — понять требования к UI/UX.
4. `animation-spec.md` — понять поведение анимаций.
5. `deploy.md` — понять как выкатывать.

## Правила обновления

- При изменении требований обновлять `spec.md`.
- При смене технологий обновлять `tech-stack.md` и `architecture.md`.
- Перед деплоем сверяться с `deploy.md`.
