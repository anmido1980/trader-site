## Этап 3. Актуальные рыночные данные (неделя 2–3)

### 3.1 Источники данных

Варианты:

- **Tinkoff Invest API** — бесплатно, MOEX + СПБ, фьючерсы, облигации.
- **MOEX ISS API** — официальное, бесплатно, только российский рынок.
- **Yahoo Finance API** — международные акции и ETF.
- **TradingView widget** — простой embed для графиков/котировок.

### 3.2 Что обновлять автоматически

- [ ] Текущие котировки в Hero/таблице.
- [ ] Курсы основных валют (USD/RUB, EUR/RUB).
- [ ] Индекс Мосбиржи.
- [ ] Дату последнего обновления данных.

### 3.3 Как обновлять

- [ ] Сделать npm-скрипт `npm run fetch-market`, который запрашивает API и записывает `src/data/market.json`.
- [ ] Запускать вручную перед публикацией или через GitHub Actions по расписанию.
- [ ] Альтернатива: client-side виджет TradingView — обновляется сам, не требует сборки.

## Этап 4. Производительность и SEO (неделя 3)

### 4.1 Производительность

- [ ] Оптимизировать все изображения в `public/images/` и `assets/images/`.
- [ ] Перевести PNG/JPG в WebP/AVIF.
- [ ] Рассмотреть `output: 'static'` вместо `output: 'server'` — сайт станет быстрее, если не нужен SSR.
- [ ] Проверить через PageSpeed Insights: цель — 90+ на мобильном и десктопе.

### 4.2 SEO

- [ ] Добавить `<meta name="description">`, Open Graph, Twitter Cards в `src/layouts/Layout.astro`.
- [ ] Добавить `robots.txt` и `sitemap-index.xml`.
- [ ] Настроить canonical URL.
- [ ] Добавить Schema.org разметку для Organization / Person.

### 4.3 Доступность

- [ ] Проверить контрастность текста.
- [ ] Добавить `aria-label` кнопкам и ссылкам без текста.
- [ ] Убедиться, что навигация работает с клавиатуры.
