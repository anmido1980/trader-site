# Спецификация секций и контента

## Общие требования

- **Тип сайта:** Лендинг (одностраничный)
- **Язык контента:** Русский
- **Адаптивность:** Mobile-first, 3 breakpoints
  - Mobile: < 768px
  - Tablet: 768px – 1024px
  - Desktop: > 1024px
- **Максимальная ширина контента:** 1200px, центрировано
- **Типографика:** Системные шрифты или 1 веб-шрифт (Inter / Manrope)
- **Цветовая схема:** Тёмная (основной фон — тёмно-синий/чёрный, акцент — зелёный для прибыли, красный для убытков, белый текст). Передаёт атмосферу трейдинга.

---

## Секция 1: Hero

**Цель:** Захватить внимание, попасть в боль целевой аудитории, показать главный оффер.

### Контент

- **Фоновое изображение/видео:** Абстрактная анимация графика (свечи, линия цены, динамический фон)
- **Заголовок (H1):** «Результаты, которые говорят сами за себя» (placeholder — Антон уточнит)
- **Подзаголовок:** «Реальная статистика торговли фьючерсами, опционами и акциями на ММВБ и мировых биржах»
- **Ключевые метрики (анимированные счётчики):**
  - Доходность за год: +XX%
  - Количество сделок: XXX
  - Win rate: XX%
  - Средний профит на сделку: +XX пунктов
- **CTA-кнопка:** «Посмотреть результаты» → скролл к секции Польза

### Поведение

- На загрузке: анимация появления текста (fade-in + slide-up), затем анимация счётчиков
- Фон: частицы или динамический градиент через Canvas/CSS

## Секция 2: Польза

**Цель:** Объяснить, почему клиенту важно следить за результатами и использовать сигналы.

### Подсекция 2.1: «Прозрачная статистика»

- **Иконка:** 📊
- **Заголовок:** «Каждая сделка — на виду»
- **Текст:** «Публикую полную статистику: вход, выход, результат, обоснование. Без скрытых убытков, без подтасовок.»
- **Визуал:** Мини-таблица последних 3-5 сделок (из JSON-данных)

### Подсекция 2.2: «Проверенные стратегии»

- **Иконка:** 🧪
- **Заголовок:** «Тестирую перед торговлей»
- **Текст:** «Каждая стратегия проходит бэктест на исторических данных. Смотрите результаты тестов — и принимайте решение на основе цифр, а не обещаний.»
- **Визуал:** Карточки с результатами бэктестов (стратегия, период, доходность, макс. просадка)

### Подсекция 2.3: «Разные инструменты»

- **Иконка:** 📈
- **Заголовок:** «Фьючерсы, опционы, акции, облигации»
- **Текст:** «Торгую на ММВБ и через Interactive Brokers. Диверсификация по инструментам и рынкам — в результатах.»
- **Визуал:** Иконки/логотипы бирж и инструментов

### Подсекция 2.4: «Обучение через наблюдение»

- **Иконка:** 🎓
- **Заголовок:** «Смотрите, как торгует профессионал»
- **Текст:** «Разбор каждой сделки: почему вошёл, почему вышел, что получил. Учитесь на реальных примерах.»
- **Визуал:** Пример разбора сделки (скриншот + пояснение)

### Поведение

- Каждая подсекция появляется при скролле (fade-in + slide-up)
- Карточки — при наведении лёгкий подъём (transform: translateY(-4px) + box-shadow)

## Секция 3: Доверие

**Цель:** Доказать, что результатам можно верить.

### Подсекция 3.1: «Реальные кейсы»

- Карточки кейсов (2-3 штуки):
  - Период
  - Инструмент
  - Стратегия
  - Результат (% доходности)
  - Ссылка на подробный разбор
- Данные из JSON

### Подсекция 3.2: «Отзывы»

- Карусель/слайдер отзывов (3-5 штук):
  - Имя
  - Роль (трейдер, инвестор)
  - Текст отзыва
  - Фото (placeholder)
- Данные из JSON

### Подсекция 3.3: «Партнёры и инструменты»

- Логотипы брокеров, бирж, платформ
- ММВБ, Interactive Brokers, QUIK, TradingView и т.д.
- Статичные логотипы, hover — подсветка

### Поведение

- Кейсы: scroll-triggered fade-in с каскадной задержкой
- Отзывы: автопрокрутка или свайп
- Логотипы: плавное появление ряда

## Секция 4: CTA (Call to Action)

**Цель:** Конвертировать посетителя в подписчика/клиента.

### Контент

- **Заголовок:** «Получайте сигналы и разборы первым» (placeholder)
- **Подзаголовок:** «Подпишитесь и получайте уведомления о новых сделках, тестах стратегий и аналитике»
- **Форма:**
  - Имя (текстовое поле)
  - Email (текстовое поле с валидацией)
  - Telegram (опционально, текстовое поле)
  - Кнопка «Подписаться»
- **Альтернативный CTA:** Кнопка «Написать в Telegram» → ссылка на Telegram Антона

### Поведение

- Фон: градиент с акцентным цветом
- Форма: inline-валидация, анимация отправки
- После отправки: сообщение об успехе (без перезагрузки)
- Форма отправляется через Formspree / Getform / или Astro API endpoint

## Секция 5: Контакты / Футер

**Цель:** Служебная информация, навигация, контакты.

### Контент

- **Навигация:** Якорные ссылки на все секции
- **Контакты:**
  - Email
  - Telegram
  - Телефон (опционально)
- **Соцсети:** Ссылки с иконками
- **Правовая информация:**
  - Дисклеймер: «Торговля на бирже сопряжена с рисками. Прошлые результаты не гарантируют будущую доходность.»
  - Политика конфиденциальности
  - © Год, имя

### Поведение

- Футер — статичный, без анимаций
- Навигация — плавный скролл к секциям (Lenis)

## Общие UI-паттерны

| Элемент                | Поведение                                              |
| ---------------------- | ------------------------------------------------------ |
| Кнопки                 | Hover: цвет акцента + scale(1.02), transition 200ms    |
| Карточки               | Hover: translateY(-4px) + box-shadow увеличение        |
| Ссылки                 | Hover: underline + цвет акцента                        |
| Изображения            | lazy loading, aspect-ratio, placeholder при загрузке   |
| Скролл                 | Плавный (Lenis), snap к секциям опционально            |
| Навигация (если fixed) | Прозрачная → полупрозрачная при скролле, backdrop-blur |

## Данные для заполнения (JSON)

Все числовые данные и списки (сделки, кейсы, отзывы) — в JSON-файлах. В спецификации указаны placeholder-значения. Антон заменит их на реальные при запуске.
