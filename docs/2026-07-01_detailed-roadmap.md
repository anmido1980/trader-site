# Детальный план развития сайта trader-site

## Как работаем

- Каждый шаг сохраняется в файле / коммите.
- После каждого блока — короткий отчёт и переход к следующему.
- Все изменения только внутри `d:\Yandex.Disk\Project\Claude\Site`.
- Перед пушем: `npm run check` и `npm run build` должны проходить.

---

## Блок 1. Подготовка и скорость (1–2 дня)

Цель: убрать технический overhead, ускорить сайт, упростить дальнейшую работу.

### Шаг 1.1. Перевести Astro на статичный вывод

**Почему:** сейчас `output: 'server'`, а сайт — лендинг без динамики. SSR лишний, замедляет отклик.

**Что делаем:**
1. Открыть `astro.config.mjs`.
2. Заменить `output: 'server'` + `adapter: vercel()` на `output: 'static'`.
3. Удалить импорт `@astrojs/vercel`.
4. В `package.json` пока оставить `@astrojs/vercel` — удалим позже, если всё ок.
5. Запустить:
   ```bash
   npm run check
   npm run build
   ```
6. Убедиться, что `dist/` содержит статичные HTML/CSS/JS.
7. Закоммитить и запушить.

**Результат:** сайт отдаётся с CDN, первое время загрузки уменьшится.

---

### Шаг 1.2. Оптимизировать изображения

**Почему:** много PNG/JPG в `public/images/` и `assets/images/`. Тяжёлые файлы замедляют загрузку.

**Что делаем:**
1. Перевести все изображения в `public/images/` в WebP:
   - hero-картинка уже WebP;
   - остальные конвертировать через `sharp-cli`.
2. Исходники PNG/JPG переложить в `assets/images/`.
3. Обновить ссылки в компонентах с `.png`/`.jpg` на `.webp`.
4. Проверить `npm run build`.
5. Закоммитить и запушить.

**Результат:** уменьшение веса страницы в 3–10 раз.

---

### Шаг 1.3. Проверить скорость

**Что делаем:**
1. Открыть `anmido.io` в режиме инкогнито.
2. В DevTools → Network → смотреть общий вес и время загрузки.
3. Проверить на [pagespeed.web.dev](https://pagespeed.web.dev/).
4. Зафиксировать результат в `docs/performance.md`.

**Результат:** понимаем базовую скорость до контентных изменений.

### Шаг 8.2. Robots и sitemap

**Что делаем:**
1. Создать `public/robots.txt`:
   ```
   User-agent: *
   Allow: /
   Sitemap: https://anmido.io/sitemap-index.xml
   ```
2. Установить `@astrojs/sitemap`.
3. Настроить в `astro.config.mjs`:
   ```js
   import sitemap from '@astrojs/sitemap';
   export default defineConfig({
     site: 'https://anmido.io',
     integrations: [sitemap()],
   });
   ```
4. Проверить `npm run build`.
5. Закоммитить и запушить.

**Результат:** поисковики видят сайт правильно.
